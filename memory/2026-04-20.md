# 2026-04-20 — Session Notes

## What We Did

### Camera Playback Investigation
- Olin mentioned camera playback freezes and that "movement stuff stopped working on the 17th"
- Confirmed via `/api/timeline` API: Apr 17+ all segments are `flags=Timing` only, Apr 16 had both `Event` and `Timing`
- Motion detection config on DVR shows `Enable=true` and `RecordEnable=1` for all 4 channels — so the *detection* is on, but the *recording schedule* was wrong
- Recording schedule had `TimeSection` mask=1 (continuous only) instead of mask=3 (continuous + motion)
- Attempted to fix via Dahua CGI `setConfig` API — discovered this firmware drops TCP connections on any URL with a space, making it impossible remotely
- Olin fixed it manually via the DVR UI

### DVR `setConfig` Deep Dive
- Tried: curl with `+`, `%20`, raw socket with literal space, Python urllib, browser fetch — all failed
- Root cause: XVR5104C-X1 firmware silently closes connection on setConfig requests before even issuing auth challenge
- getConfig works fine via digest auth with qop=auth + nc/cnonce

### Timeline UX Improvements
- Scroll zoom was only enabled in clip mode — enabled it in normal playback mode too
- Added pan-by-drag when zoomed in (grab cursor, 5px threshold to distinguish from click)
- Click still seeks; drag pans; double-click resets zoom
- Clip mode: changed default from 30s window to 10-minute centered window
- Clip mode: timeline auto-zooms to clip window when opened so handles are visible and grabbable

### Conversations
- Olin asked about FTP server for clips — decided against it, on-demand playback is sufficient for "something happened, let me go check" use case
- Olin asked how OpenClaw maintains memory — explained the file-based system, noted I hadn't been writing things down, created MEMORY.md and daily notes

## To Remember

- Physical access or Lorex app needed to change DVR recording schedule if it resets again

## Afternoon Session (continued)

### Weather / Radar / Wind / Route — New Features Built
All four new features built and deployed to the dashboard:

**WindWidget** (`src/components/WindWidget.tsx`):
- Compact card on Home + Climate showing Leaflet map with wind arrows (Windfinder-style)
- Arrows color-coded by speed (cyan/green/amber/red), speed labels per point
- Tap opens detail overlay: wind map + current stats + 24h hourly chart with direction arrows
- Also added as Wind overlay FAB on Map tab
- Data from Open-Meteo free API (no key needed), refreshes every 15 min

**RadarWidget** (`src/components/RadarWidget.tsx`):
- Compact card on Home + Climate with embedded Leaflet map + RainViewer radar tiles
- Starts **paused at current time** (not animating) — play button to animate through frames
- Stops at end of frames instead of looping (was causing jank/resets)
- Map overlay FAB on Map tab overlays radar tiles directly on the Leaflet vanlife map
- Fixed: was using `window.L` for map overlay — switched to imported L

**RoutePreview** (`src/components/RoutePreview.tsx`):
- Small Leaflet map on Home page showing last 48h route from vanlife API
- Tap navigates to Map tab
- Fixed reload bug: `currentLat/currentLon` were in useEffect deps, causing full re-init every GPS update — removed from deps, init runs once on mount
